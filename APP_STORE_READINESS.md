## 1. Architecture decision (do this FIRST)

🚧 **You have not decided how Capacitor will load the app.**

Today the codebase ships a server-rendered Next 16 app (force-dynamic pages, middleware, API routes, Supabase SSR, Resend, Stripe webhooks). Capacitor expects a folder of static files at `webDir: 'out'`. **Those two world-views are incompatible** until you choose one of:

- **A. Remote-URL wrap (recommended, fast path).** Deploy the Next.js app to Vercel/Render/Netlify. Update `capacitor.config.ts` to point `server.url` at that deployment. The native shell becomes a thin WebView. ✅ Keeps every feature working. ❌ App is unusable offline beyond the SW cache. ❌ App Store reviewers sometimes flag "this is just a website" — mitigate with deep platform integrations (push, share-intake, biometrics).
- **B. Static SPA + hosted API.** Move all server logic to API routes hosted separately, refactor pages to client-rendered, set `output: 'export'` in `next.config.ts`, ship `/out` via Capacitor. Significant refactor. Loses SSR speed. Gains offline-first.

**Decide before any other App Store work — it changes everything below.**

## 11. Stripe / payments — Apple-specific concern

Apple's IAP rules require that digital subscriptions (like ManyHandz access) **must use Apple In-App Purchase** when sold inside an iOS app. **You cannot use Stripe for iOS subscriptions** unless ManyHandz qualifies as a "reader app" or you use the new "external payment link" entitlement.

Options:
- **A. Use StoreKit / RevenueCat for iOS subscriptions, Stripe for web + Android.** Most common pattern. RevenueCat unifies the SDKs. Pricing must match across platforms ±15% to satisfy Apple.
- **B. Apply for the External Purchase Link entitlement.** Available since 2024; Apple takes 27% commission instead of 30%, and you must show a scare-screen disclosure.
- **C. Make the iOS app free-to-use (no subscription in the app), force users to subscribe on the web.** Allowed if you don't even *mention* paid features in the app — risky and limits monetization.

**Decide this before submitting.** Today the Stripe flow is wired up but iOS will reject as-is.
