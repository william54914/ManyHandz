Household modes
The problem

Every household has an invisible weight nobody is tracking.

Chores fall on the same shoulders week after week. Promises about who-owes-whom evaporate. Kids and roommates argue about fairness with no data to back it up. ManyHandz fixes this with a single, unified system that tracks contribution, automates rotation, and rewards consistency.

⚖️

Invisible inequality

The dishes get done — but always by the same person. Without measurement, the household never sees the imbalance until someone explodes about it.

📋

Lost accountability

Sticky notes, shared docs, group texts, and one-off reminders all fall through the cracks. Nothing connects "what was promised" to "what got done."

💸

Forgotten IOUs

Allowances, splits, and "I owe you a treat" never get tracked. Money and favors are lost in the shuffle, eroding trust over time.

One app. Every household. Real fairness.

ManyHandz is the first chore-coordination platform that combines auto-rotation, AI-verified photo proof, gamification, and a complete ledger for promises — money and non-money alike.

Mode-aware UX

Two completely different apps in one codebase.

ManyHandz adapts every screen, color, animation, and permission to who's using it. Configuration-driven — adding a third mode (Office, College Dorm, Care Team) is a single file.

👨‍👩‍👧‍👦 Family mode

Parents manage, kids earn.

Playful, gamified, age-appropriate. Parents set up chores and approve completions. Kids see big level-up celebrations, work toward goals like a new video game, and unlock badges. Allowance flows through Settle Up.

  • Parent + Kid role permissions matrix
  • Approval queue with photo verification
  • Achievement badges, levels (1–50+), XP
  • Big-ticket goals with auto-contribute
  • Rewards marketplace with points
  • Confetti, celebrations, streak fire
  • Birthday Pass & gift points
  • Letter-grade weekly report cards
🏠 Roommate mode

Equal housemates, fair split.

Clean, minimal, adult-oriented. Everyone has equal permissions. No approvals, no gamification clutter — just transparent fairness scoring, auto-rotation, and a complete ledger of who owes whom.

  • Honor system — instant completion
  • Fairness scoring is the hero feature
  • Auto-rotation: round-robin or fixed
  • Photo proof (optional)

Security & compliance

Production-grade hardening on day one.

App Store-ready privacy posture. Defense-in-depth from edge headers down to database-level RLS.

  • Row-Level Security on every Postgres table. The database itself enforces "users can only see their household's data." No app-layer bypass possible.
  • HSTS + preload with 2-year max-age. Every response forces HTTPS, includes subdomains.
  • Content-Security-Policy with strict default-src 'self', allow-list for Stripe + Supabase only.
  • X-Frame-Options: DENY — no clickjacking via iframe embedding.
  • Permissions-Policy blocks microphone & geolocation. Camera enabled for self only (photo proof).
  • Referrer-Policy: strict-origin-when-cross-origin — protects user privacy on outbound links.
  • WebAuthn / passkeys — biometric login. No password-reuse risk.
  • Webhook signature verification — Stripe + Supabase webhooks rejected if signature invalid.
  • CRON_SECRET-protected cron endpoints — only the scheduler can trigger background jobs.
  • Service-role isolation — admin DB writes never run in client code, only on the server.
  • Zod validation on every API input. Type-narrow at the boundary.
  • Signed URLs (60s TTL) for private photo storage — proof & reference photos.
  • Stripe Checkout / Customer Portal — card data never touches our servers. PCI scope minimized.
  • Account deletion flow — CCPA / GDPR / Apple App Store §5.1.1(v) compliant.
  • Email confirmation enforced — unverified accounts can't perform sensitive actions.
  • Path-scoped storage — every uploaded file at `{household_id}/...` so RLS-equivalent isolation applies.
